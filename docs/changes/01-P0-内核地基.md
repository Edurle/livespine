## 一句话总结

搭建纯数学内核(`lp-core`),实现 2D 骨骼动画的变换/蒙皮基础,用黄金值测试锁死正确性。退出标准:`livepine solve` 能打印顶点坐标且与手算一致。**不含任何渲染。**

## 做了什么

### 1. 初始化 Rust workspace

- 顶层 `Cargo.toml`(workspace,3 个成员)
- workspace 依赖统一管理(serde / serde_json)

### 2. 实现 `lp-core`(5 个模块)

| 模块 | 文件 | 内容 | 规范来源 |
|---|---|---|---|
| math | `math.rs` | `Vec2`/`Affine`/`BoneLocal` + `local_to_affine`/`multiply`/`invert`/`world_rotation`/`world_scale` | 04-实现 §1~4 |
| skeleton | `skeleton.rs` | 骨骼树 + `update_world`(前序复合) + `precompute_bind_inverse` + 前序不变式校验 | 04-实现 §3, §5 |
| attach | `attach.rs` | `Vertex` + `RegionAttachment` + 权重校验 | 04-实现 §4 + 11 |
| skin | `skin.rs` | LBS 蒙皮:`deform_matrix`/`skin_vertex`/`skin_region` | 04-实现 §5 |

### 3. 实现 `lp-io`

- 最小 `.lp`(JSON)读写:`LpFile`/`SkeletonDef`
- 基础校验(format 字段、权重和)

### 4. 实现 `lp-cli`

- `solve` 子命令:加载 → `update_world` + 蒙皮 → 打印骨骼 world + 顶点坐标
- 满足 P0 退出标准

### 5. 测试

- **18 个单元测试**全绿(覆盖仿射运算、父子复合、旋转、shear、bind逆、蒙皮)
- **3 个黄金值用例**(`hand-computed` 基准):
  - `transforms/parent_chain`(父子平移复合)
  - `transforms/rotated_chain`(旋转下父子复合)
  - `skinning/two_bone_blend`(两骨 0.5/0.5 权重混合)
- **黄金值加载器**(`tests/golden.rs`):自动遍历用例目录、加载、比对、报告

## 关键决策

1. **不引入外部数学库**(glam/nalgebra):P0 数值简单,自实现保持内核零依赖。呼应 `03-分层架构` 原则 1。
2. **每顶点最大骨骼数 4**:定义 `MAX_WEIGHTS = 4`,预留 8 能力。
3. **region 附件用 4 矩形角顶点**:P0 不做复杂三角化(mesh 化留 P1)。
4. **serde 算"零依赖"例外**:派生宏是编译期,非运行时依赖。
5. **P0 不做 dirty 增量更新**:每帧全量 `update_world`,dirty 子树优化留 P3。

## 过程教训

**黄金值基准值算错一次**:蒙皮测试 `single_bone_weight_identity_at_bind` 首次失败,根因是**我手算的期望值错**(误把"顶点 local 是骨骼局部坐标"当成"世界坐标")。修正后通过。

这正是 `12a-黄金值测试-概念` 强调的陷阱:**黄金值的基准值本身可能错,建立基准时必须仔细推导**。该教训已记入 Obsidian `14-P0内核实现清单` 验收区。
